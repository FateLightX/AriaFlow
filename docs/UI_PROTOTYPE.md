# AriaFlow UI Prototype

状态：线框稿
版本：v0.1
最后更新：2026-07-09
参考：`Design/主页面.png` 作为气质参考，不严格复刻

## 1. 设计目标

AriaFlow 第一版 UI 目标是做一个更接近 macOS 原生工具的下载管理器，而不是复刻 Motrix Next 的 Web/Tauri 界面。

核心原则：

- 原生结构优先：使用 toolbar、sidebar、list/table、inspector、sheet、settings window。
- 信息密度适中：下载任务是主内容，控制按钮和状态信息只占必要空间。
- 状态明确：连接失败、空列表、下载中、磁力元数据等待都必须有清楚反馈。
- 少做装饰：颜色主要依赖系统语义色，只保留 AriaFlow 蓝色作为选中和主操作强调。
- 服务 SwiftUI 实现：原型不追求 Figma 级视觉精度，重点锁定布局、状态和交互。

## 2. 设计系统

### 2.1 颜色

优先使用 macOS 系统语义色，避免自定义主题系统。

| Token | 用途 | 建议 |
| --- | --- | --- |
| `Accent` | 主操作、选中状态 | AriaFlow Blue `#2F7DFF` |
| `Background` | 主窗口背景 | `NSColor.windowBackgroundColor` |
| `Sidebar` | 侧边栏背景 | `NSColor.controlBackgroundColor` |
| `Separator` | 分割线 | `NSColor.separatorColor` |
| `TextPrimary` | 主要文字 | `NSColor.labelColor` |
| `TextSecondary` | 辅助文字 | `NSColor.secondaryLabelColor` |
| `Success` | 完成/连接成功 | `NSColor.systemGreen` |
| `Warning` | 等待/暂停 | `NSColor.systemOrange` |
| `Error` | 失败/连接断开 | `NSColor.systemRed` |

深色模式不单独设计主题，只要求系统语义色自然适配。

### 2.2 字体

使用系统字体：

- 标题：System Semibold 15-17。
- 列表主文本：System Regular 13-14。
- 状态和辅助信息：System Regular 11-12。
- 数字速度/大小：Monospaced Digit，可用于速度、百分比、大小对齐。

### 2.3 图标

使用 SF Symbols：

| 操作 | Symbol |
| --- | --- |
| 添加 | `plus` |
| 开始/继续 | `play.fill` |
| 暂停 | `pause.fill` |
| 删除 | `trash` |
| 设置 | `gearshape` |
| 全部 | `tray.full` |
| 下载中 | `arrow.down.circle` |
| 等待中 | `clock` |
| 已完成 | `checkmark.circle` |
| 已失败 | `xmark.circle` |
| 历史 | `clock.arrow.circlepath` |
| 详情 | `sidebar.right` |

## 5. Sidebar

Sidebar 是主筛选入口，不再使用顶部横向筛选栏。

```text
下载任务
  全部                         8
  下载中                       2
  等待中                       1
  已完成                       4
  已失败                       1

资料库
  历史
```

交互：

- 点击筛选项切换 `TaskFilter`。
- 计数来自当前任务集合和历史集合。
- 当前筛选项使用 Accent 背景或系统 sidebar selection 样式。
- `历史` 独立显示完成/失败记录，不影响 aria2 当前 session。

## 6. Toolbar

```text
AriaFlow                          [+] [▶] [Ⅱ] [🗑]     [详情] [⚙]
```

按钮行为：

| 按钮 | 可用条件 | 行为 |
| --- | --- | --- |
| 添加 | 总是可用 | 打开 Add Task Sheet |
| 继续 | 有选中 paused/waiting/error 可重试任务 | 对选中任务执行继续 |
| 暂停 | 有选中 active/waiting 任务 | 对选中任务执行暂停 |
| 删除 | 有选中任务 | 打开删除确认 |
| 详情 | 有选中任务 | 显示/隐藏 inspector 或打开详情 sheet |
| 设置 | 总是可用 | 打开 Settings Window |

默认情况下，没有选中任务时继续、暂停、删除、详情置灰。

| 下载 | 上传限速 | text field | `不限` | 支持空值或速度值。 |
| 通知 | 下载完成通知 | toggle | 开启 | 任务完成时发送系统通知。 |
| 通知 | 下载失败通知 | toggle | 开启 | 任务失败时发送系统通知。 |
| 通知 | 任务开始通知 | toggle | 关闭 | 任务进入下载状态时发送系统通知。 |

| 引擎 | 打开日志 | button | - | 打开本机引擎日志位置或日志窗口。 |

### 11.7 交互规则

- 设置项修改后即时写入本地设置模型，不需要手动保存。
- 默认保存位置使用系统目录选择器；用户取消选择时保持原值。
- RPC 端口修改后在当前行或底部显示“重启引擎后生效”提示。
- `重试连接` 只触发引擎重连，不关闭设置窗口。
- Settings window 不承担任务创建、任务删除或下载详情展示。

## 12. Delete Confirmation

```text
删除任务？

这会从 AriaFlow 中移除 2 个任务。已下载的文件会保留在磁盘上。

[ ] 同时删除本地文件

[取消] [删除任务]
```

规则：

- 默认不勾选“同时删除本地文件”。
- 勾选后主按钮文案改为“删除任务和文件”。
- 对已完成任务和下载中任务使用同一确认模型。

## 13. Status Bar

状态栏只属于主内容区，不横跨 Sidebar，也不放在 Inspector 下方。Sidebar 保持完整高度；Inspector 保持独立详情区。

```text
● 已连接   ↓ 12.4 MB/s   ↑ 128 KB/s   2 个下载中   1 个等待中   4 个已完成   1 个已失败
```

状态：

| ConnectionState | 显示 |
| --- | --- |
| `starting` | `● 正在连接`，灰色或橙色 |
| `connected` | `● 已连接`，绿色 |
| `failed` | `● 连接失败`，红色 |
| `stopped` | `● 已停止`，灰色 |

```text
上传速度 128 KB/s
────────────
退出 AriaFlow
```

规则：

- 第一版菜单栏只描述入口和状态。
- 菜单栏速度可由设置开关控制。
- 不做复杂图形化菜单。

### 14.2 Dock

第一版 Dock 行为：

- 点击 Dock 图标显示主窗口。
- 下载中可显示简单 badge：活动任务数或速度，最终实现二选一。
- Dock progress 可后置；如果实现成本低，可显示整体完成比例。

## 15. UI State Model

### 15.1 ConnectionState

```text
starting -> connected
starting -> failed
connected -> failed
failed -> starting
connected -> stopped
stopped -> starting
```

| State | UI |
| --- | --- |
| `starting` | 内容区显示连接中，toolbar 只允许设置 |
| `connected` | 显示任务列表或空状态 |
| `failed` | 显示连接失败状态 |
| `stopped` | 显示已停止状态，可重试连接 |

### 15.2 TaskFilter

| Filter | 内容 |
| --- | --- |
| `all` | 当前 session 内所有任务 |
| `active` | 下载中和做种中 |
| `waiting` | 等待中和暂停 |
| `complete` | 已完成 |
| `failed` | 失败 |
| `history` | 本地历史记录 |

### 15.3 TaskRow

字段：

- 名称。
- 状态。
- 进度。
- 已完成大小。
- 总大小。
- 下载速度。
- 上传速度。
- 剩余时间。
- 协议。

### 15.4 TaskDetail

字段：

- 基础信息。
- 保存路径。
- 文件列表。
- 错误信息。
- BT 摘要。
- ED2K 摘要。

### 15.5 GlobalStats

字段：

- 下载速度。
- 上传速度。
- 活动任务数。
- 等待数。
- 完成数。
- 失败数。

## 16. 交互流

### 16.1 添加 URL

```text
点击添加
-> 打开 Add Task Sheet
-> 输入 URL
-> 选择保存目录
-> 开始下载
-> 关闭 Sheet
-> 任务出现在列表
```

### 16.2 添加 Torrent

```text
点击添加
-> 选择 Torrent 文件
-> aria2 读取文件列表
-> 打开 File Selection Sheet
-> 选择文件
-> 开始下载
```

### 16.3 添加 Magnet

```text
输入 magnet
-> 添加 metadata 任务
-> 内容区或 Sheet 显示“正在读取文件列表”
-> 元数据完成
-> 打开 File Selection Sheet
-> 选择文件
-> changeOption(select-file)
-> unpause
```

### 16.4 查看详情

```text
选中任务
-> Inspector 更新
-> 点击详情按钮隐藏/显示 Inspector
-> 窄窗口双击任务打开详情 Sheet
```

### 16.5 删除任务

```text
选中任务
-> 点击删除
-> 删除确认
-> 默认只删除任务记录
-> 可选同时删除本地文件
```

## 18. v1 保留与延后

### v1 保留

- 原生 toolbar + sidebar + content + inspector。
- 连接中、连接失败、空任务、任务列表状态。
- 添加 URL、torrent、magnet。
- 文件选择 sheet。
- 基础设置。
- 菜单栏入口。
- 简单通知。

### v1 延后

- thunder 转换。
- 完整 peer/tracker 面板。
- 自动更新。
- 多语言。
- 主题系统。
- 复杂菜单栏图形状态。
- 高级请求头、Cookie、Referer、代理完整 UI。
- Dock progress 精细视觉。

## 19. 设计自检

- 没有沿用顶部横向筛选栏，改为更 macOS 原生的 sidebar。
- 没有引入卡片堆叠或营销式布局。
- 没有设计主题系统，避免第一版范围膨胀。
- 没有为 peer/tracker、扩展能力和高级下载参数提前占满主界面。
- 任务列表、连接状态和添加任务仍是第一屏核心。
